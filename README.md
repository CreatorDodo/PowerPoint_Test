# 사전 과제 프로젝트 마일스톤 및 이슈, 방향성

## 전체 방향성 개요
1. **UI 일치**  
   - 샘플 페이지의 리본 탭(Ribbon Tab)과 버튼, 슬라이드 영역의 배치, 색상, 폰트, 숨김/고정 모드 전환 등 UI가 최대한 동일하게 보이도록 구현합니다.  
   - 전역 폰트로 Pretendard를 사용합니다.

2. **핵심 기능**  
   - **슬라이드 영역 확대/축소**  
     - Ctrl(Cmd)+마우스 스크롤 또는 하단 슬라이더를 통한 확대/축소 기능.  
   - **리본 탭 고정/해제 모드**  
     - 탭 UI를 항상 표시하거나, 필요할 때만 표시하는 두 가지 모드 전환.  
     - 탭 제목 더블클릭으로 모드 전환(고정 ↔ 자동 숨김).  
   - **도형(직사각형/원) 추가 및 드래그**  
     - 위치·크기·색깔·테두리(3px) 모두 랜덤 생성.  
     - 마우스로 드래그 이동 가능.  
   - **이미지 파일(PNG, SVG) 추출**  
     - 슬라이드 영역에 표시된 내용만 캡처하여 PNG/SVG로 저장.

3. **기술 스택과 구조**  
   - React 18 이상 또는 Next.js, Vite 등 React 기반 프레임워크 자유롭게 선택.  
   - 도형 및 슬라이드 영역 관리는 Canvas/DOM/fabric.js/konva.js 등 원하는 방식으로 구현 가능(필수 요구 사항만 충족하면 OK).  
   - HTML 요소를 Canvas로 변환해주는 라이브러리(html2canvas, dom-to-image, Konva 등) 활용 가능.

## 마일스톤 및 주요 이슈

아래는 한 가지 예시 플랜입니다. “언제까지 어느 범위의 기능을 완성한다”는 식으로 마일스톤을 잡아두고, 구현 과정에서 발생하는 세부 이슈들을 등록/관리하는 식으로 진행합니다.

### 마일스톤 1: 프로젝트 셋업 및 기본 레이아웃
- **목표**: 기본 폴더 구조, 빌드 환경, 전역 폰트(Pretendard) 세팅, 리본 탭 및 슬라이드 영역의 뼈대 UI 구현.
- **주요 작업**
  1. **React 프로젝트 초기 세팅**  
     - CRA(Create React App), Vite, Next.js 등 선택 후 셋업.  
     - Pretendard 폰트 글로벌 적용(CSS import 혹은 폰트 파일 직접 호스팅).
  2. **전체 레이아웃 구성**  
     - 상단: 로고/프로젝트명(Duyo Test 등), 리본 탭 메뉴 “파일·홈·삽입·슬라이드쇼·도움말”.  
     - 중앙: 슬라이드 표시 영역(빈 박스), 향후 도형을 올려둘 캔버스/DOM.  
     - 하단: 확대/축소 슬라이더 위치.
  3. **리본 탭 기본 스타일링**  
     - 탭 간 전환 시 각 탭마다 다른 메뉴(예: 파일 탭 클릭 → “열기·저장·SVG/PNG 내보내기” 등) 노출.  
     - 탭 숨김/고정 모드 전환은 아직 동작 안 해도 OK. 우선 자리를 잡아두기.

- **예상 이슈**  
  - 리본 탭을 단일 컴포넌트 vs. 탭별 컴포넌트로 분리할지 결정.  
  - CSS(또는 styled-components/Sass 등)로 레이아웃을 어떻게 구조화할지.  
  - 폰트 적용 시 브라우저 호환성 이슈가 있는지 점검.

---

### 마일스톤 2: 리본 탭 고정/해제 기능 & 슬라이드 확대/축소
- **목표**: 리본 탭을 더블클릭으로 숨기는 기능과, 슬라이드 확대/축소(Zoom) 기능을 구현해 UI 조작의 핵심 부분 완성.
- **주요 작업**  
  1. **리본 탭 모드 전환**  
     - “고정 모드” → 리본 탭이 항상 보임.  
     - “자동 숨김 모드” → 탭 제목 영역만 보이고, 마우스 오버·더블클릭 시 메뉴 표시.  
     - 리본 영역 감춤/노출 시 슬라이드 영역 레이아웃 자동 재조정.
  2. **슬라이드 줌(Zoom) 기능**  
     - Ctrl(Cmd)+마우스 스크롤 이벤트로 확대/축소.  
     - 하단 슬라이더(또는 +/– 버튼)로도 확대/축소 가능.  
     - 실제 구현은 Transform(scale) 사용, Canvas scale 변경, 혹은 CSS Zoom 등 적절히 선택.

- **예상 이슈**  
  - 이벤트 분기(Ctrl 키 + 스크롤)를 OS·브라우저별로 정확히 구분 처리.  
  - 슬라이드 및 도형 드래그 기능과 Zoom 기능 간 충돌(좌표계 변환) 문제.  
  - 모드 전환 시 슬라이드 크기가 변해도 도형 배치가 깨지지 않도록 처리할지 여부(또는 화면 레이아웃만 조정).

---

### 마일스톤 3: 도형(직사각형·원) 생성 및 드래그
- **목표**: “삽입” 탭에서 직사각형 또는 원을 무작위 색상/크기로 생성하고, 생성된 도형을 드래그 이동 가능하게 함.
- **주요 작업**
  1. **도형 생성 로직**  
     - `삽입 > 직사각형` 클릭 → 랜덤 너비/높이, 랜덤 배경색, 테두리 3px(색상도 랜덤) 지정.  
     - `삽입 > 원` 클릭 → 랜덤 반지름, 랜덤 색상, 테두리 3px(색상도 랜덤).  
     - 도형은 슬라이드 내부에 나타나야 하므로, 시작 좌표도 랜덤(단, 슬라이드 범위 내) 설정.
  2. **드래그 기능**  
     - 마우스로 도형을 클릭·홀드 후 이동하면, 실제로 도형이 X/Y 좌표를 따라 움직이도록 구현.  
     - 확대/축소(Zoom) 상태에서도 적절히 드래그가 동작하도록 좌표 계산.
  3. **도형 레이어**  
     - 생성 순서에 따라 도형이 위/아래로 겹칠 수 있음(z-index 또는 Canvas 레이어 처리).

- **예상 이슈**  
  - DOM vs Canvas 기반에 따라 드래그 이벤트 처리 방식이 달라질 수 있음.  
  - Zoom이 적용된 상태에서 드래그 좌표를 어떻게 매핑할지.  
  - 도형이 슬라이드 밖으로 벗어나지 않도록 제한할지 여부(샘플 요구사항에 명시돼 있지 않으므로 자유롭게 결정).

---

### 마일스톤 4: 이미지 파일(PNG, SVG) 추출
- **목표**: 슬라이드 영역의 실제 내용(도형들)을 PNG 또는 SVG로 내보낼 수 있게 구현.
- **주요 작업**  
  1. **Export 기능 구현**  
     - `파일 > 내보내기 > PNG` 클릭 시 → 슬라이드 영역을 PNG로 저장.  
     - `파일 > 내보내기 > SVG` 클릭 시 → 슬라이드 영역을 SVG로 저장(가능하다면).
  2. **슬라이드 영역만 추출**  
     - UI 영역(리본 탭 등)은 제외하고, 실제 ‘슬라이드 캔버스’ 부분만 저장되도록 처리.  
     - HTML to Canvas(또는 dom-to-image 등) 라이브러리 사용 시 ‘특정 DOM 영역만’ 캡처.
  3. **후처리(불필요 효과 제거)**  
     - 브라우저 상에서 보이는 테두리, 모서리 곡선 효과(경계 라운딩)는 내보내기 시 제거.  
     - 텍스트/도형 색상만 정확히 반영되도록.

- **예상 이슈**  
  - HTML to Canvas 방식에서 CSS transform(Zoom)을 어떻게 해석할지.  
  - SVG 내보내기 시, 브라우저마다 호환성 이슈 및 폰트 링크 문제 발생 가능.  
  - “슬라이드만 캡처” 위해 슬라이드 DOM을 별도의 ref로 잡고, 그 부분만 Export 처리.

### 마일스톤 5: QA 및 폴리싱(Polishing)
- **목표**: 사소한 버그 수정, UI 디테일 조정, 문서화 등 최종 마무리.
- **주요 작업**  
  1. **버그 이슈 처리**  
     - 예: 도형 드래그 시 깜빡임, 확대/축소 배율 0% 이하로 내려가려는 문제 등  
  2. **UI 디테일 및 반응형 개선**  
     - 리본 탭이나 슬라이드 영역 크기 변경 시 레이아웃 무너짐 없도록 보완.  
  3. **사용자 안내(도움말)**  
     - ‘도움말’ 탭 클릭 시 간단한 기능 안내나 사용법 표시(필요 시).  
  4. **최적화**  
     - 도형이 많아질 때 렌더링 지연이 없는지 확인.  
     - 불필요한 re-render 줄이기(React.memo 등).

## 정리
- **방향성**: “PowerPoint 유사 UI”라는 핵심 키워드를 중심으로, 리본 탭과 슬라이드 영역 간 상호작용(고정/숨김, 확대/축소, 도형 삽입/드래그, 이미지 익스포트)에 충실합니다.  
- **마일스톤**: 대략 5단계(레이아웃 → 리본 탭/Zoom → 도형 삽입/드래그 → Export → QA 정리)로 나누어 진행하면 기능별로 명확히 관리하기 좋습니다.  
- **이슈 관리**: 각 마일스톤 안에서 구현 단위를 세분화해 Issue(또는 Task)로 만들고, 커밋/PR 단위로 연결하여 추적합니다.
